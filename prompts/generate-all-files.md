# Generate All Starting Files for a New Project

Copy this AFTER you have generated PROJECT_BRIEF.md.
Paste in the same chat. Get all 5 starting files instantly.

---

You just generated PROJECT_BRIEF.md for this project.

Now generate starter content for these 5 files
using the same project context:

1. BRAIN.md
   - Current goal: first real task
   - Blocker: none
   - Phase: Research
   - First hypothesis: your best guess about the answer
   - Biggest risk: most likely way this project fails
   - Keep under 200 words

2. SPEC.md
   - Technical rules and constraints
   - Tools and data sources available
   - Cost or time constraints
   - Non-negotiable rules
   - Keep under 200 words

3. DO_NOT_REPEAT.md
   - List 3 obvious dead ends to avoid immediately
   - Things that always fail for this type of project
   - Format as table: Date | Idea | Why Rejected | Decision Ref

4. RED_TEAM.md
   - List 5 most likely ways this project fails or produces wrong results
   - Label each: High / Medium / Low severity
   - Format as ATTACK-001, ATTACK-002 etc

5. VALIDATION.md
   - Checklist of minimum 8 items before trusting any result
   - Split into: Pre-Work Checklist and Results Checklist
   - Specific to this project type

Output each file separately with the filename as a header.
Keep every file under 200 words.
Do not add extra explanation outside the file content.
