# How to Start a New Project

Simple 5-step process. Takes 10 minutes.

---

## Step 1: Fork This Template

1. Go to github.com/vishalm111296-commits/ai-brain-os
2. Tap the green button that says Use this template
3. Tap Create a new repository
4. Name it your project
   Examples:
   - three-slap-research
   - youtube-system
   - business-plan
   - fitness-tracker
5. Set to Public
6. Tap Create repository

Done. All 10 files and all prompts are copied automatically.

---

## Step 2: Fill in SPEC.md

Open SPEC.md in your new repo.
Answer these questions:
- What are we building?
- What does success look like?
- What data or tools do we have?
- What are the rules we cannot break?

Commit the file.

---

## Step 3: Fill in BRAIN.md

Open BRAIN.md.
Fill in:
- Current goal (one sentence)
- Current blocker (or write none)
- Current best hypothesis (your best guess right now)
- Next action (the very first task)

Commit the file.

---

## Step 4: Update Your Phone Notes

Open Google Keep.
Open your START note.
Replace the raw links with your NEW repo raw links.

New raw links look like:
https://raw.githubusercontent.com/vishalm111296-commits/YOUR-NEW-REPO-NAME/main/BRAIN.md

Just replace ai-brain-os with your new repo name in every link.

---

## Step 5: Run Your First Session

Open any AI.
Paste your START note.
Change the last line to your first task.
Start working.

That is it. Your new project has a full brain from day one.

---

## Where Each Type of File Goes

| What AI generates | Where to save it |
|---|---|
| Code, scripts, Python files | code/ folder |
| Experiment results | Add row to RESULTS.csv |
| Confirmed observations | Add line to FACTS.md |
| Decisions made | Add block to DECISIONS.md |
| Rejected ideas | Add line to DO_NOT_REPEAT.md |
| Session summary | New dated file in sessions/ folder |
| Current status | Overwrite BRAIN.md |
| Stop point | Overwrite CHECKPOINT.md |
| Weekly digest | Overwrite WEEKLY_SUMMARY.md |
